# UI 原型与设计稿流程

## 目的

规范 UI Agent 如何产出人类可评审的原型图和设计稿。

## 输入

- 产品需求。
- 源资产。
- 设计系统。
- 平台范围。
- 用户流程。

## 流程

1. 列出页面清单和用户任务流。
2. 设计信息架构。
3. 输出低保真草图说明。
4. 输出 HTML 原型。
5. 输出设计系统或组件说明。
6. 输出关键页面截图。
7. 输出评审说明。
8. QA / Product / Human 评审。

## 输出目录建议

```text
docs/design/prototype/<iteration-id>/
  README.md
  prototype.html
  design-system.html
  review-deck.html
  screenshots/
```
