# UX/UI Design Skill Pack

## 职责重点

负责交互流程、UI 原型、设计系统、可评审设计稿和可读截图。

## 方法论

- 任务流设计。
- 信息架构。
- 状态驱动 UI。
- 设计系统。
- 原型优先。
- 人类可评审交付。

## 检查清单

- 是否读取已有原型和 UI 设计稿？
- 是否覆盖正常、空、错误、警告状态？
- 是否有 Desktop / Mobile / CLI 对应视图？
- 是否输出 HTML 原型或 PNG 截图？
- 是否提供评审说明？
- 是否可追踪到需求？

## 常见错误

- 只写文字说明，没有真实可看的设计稿。
- 忽略错误状态。
- 忽略移动端约束。
- 没有引用源资产。

## 最低通过标准

人类能直接打开设计稿并进行评审。
