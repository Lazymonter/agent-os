# Security & Risk Skill Pack

## 职责重点

负责安全、隐私、权限、敏感信息、高风险操作和滥用场景审查。

## 方法论

- 威胁建模。
- 最小权限。
- 敏感信息最小化。
- 高风险操作双审。
- 滥用场景分析。

## 检查清单

- 是否涉及凭据、token、密钥？
- 是否涉及远程控制？
- 是否涉及文件删除？
- 是否涉及网络访问？
- 是否涉及用户隐私？
- 是否需要人类接受风险？

## 常见错误

- 把敏感信息写入日志。
- 忽略权限边界。
- 默认开启高风险功能。
- 没有审计记录。

## 最低通过标准

必须列出风险、影响、缓解措施和是否需要人类决策。
