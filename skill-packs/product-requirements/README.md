# Product & Requirements Skill Pack

## 职责重点

负责把目标转成产品需求、用户故事、用例和验收标准。

## 方法论

- 用户目标分析。
- 需求分类：新功能、变更、缺陷、设计、实现、发布。
- MoSCoW 优先级。
- Given / When / Then 验收标准。
- 需求可测试性检查。

## 检查清单

- 需求是否有明确用户价值？
- 是否属于当前项目范围？
- 是否影响已有原型或 UI？
- 是否需要更新验收标准？
- 是否存在非功能需求？
- 是否有明确“不做什么”？

## 常见错误

- 把工程实现细节误写成产品需求。
- 用模糊词，例如“尽量”“比较好”“适当”。
- 没有定义异常流程。
- 忽略已有原型和源资产。

## 最低通过标准

需求必须可测试、可追踪、可拆解为任务。
