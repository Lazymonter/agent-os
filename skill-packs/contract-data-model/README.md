# Contract & Data Model Skill Pack

## 职责重点

负责 schema、API 契约、数据模型、兼容性和契约测试。

## 方法论

- Schema-first。
- 版本化。
- 契约测试。
- 兼容性检查。
- 示例数据驱动。

## 检查清单

- 是否涉及 schema？
- 是否需要新增字段或枚举？
- 是否破坏兼容性？
- 是否需要 human approval？
- 是否需要更新示例数据？
- 是否需要更新 golden files？

## 常见错误

- 擅自新增枚举。
- schema 与实现不一致。
- 没有版本策略。
- 没有说明破坏性变更。

## 最低通过标准

契约必须明确、可验证、可兼容或明确标记破坏性。
