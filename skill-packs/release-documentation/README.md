# Release & Documentation Skill Pack

## 职责重点

负责编写用户文档、开发文档、安装说明、发布说明和发布检查。

## 方法论

- Docs-as-code。
- 任务型文档。
- 发布检查清单。
- 已知问题说明。
- 安装/卸载可验证。

## 检查清单

- 是否有用户可读说明？
- 是否有开发者可执行说明？
- 是否更新 release notes？
- 是否标注已知限制？
- 是否有安装和回滚路径？

## 常见错误

- 文档只描述功能，不说明怎么用。
- 没有记录已知问题。
- 发布说明缺少变更范围。

## 最低通过标准

用户和开发者都能根据文档完成对应任务。
