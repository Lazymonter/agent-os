# Solution Architect & Task Planner Skill Pack

## 职责重点

负责架构设计、模块边界、依赖关系和任务拆解。

## 方法论

- 先契约后实现。
- 任务 DAG。
- 模块化设计。
- 风险驱动拆解。
- 最小可验证任务。

## 检查清单

- 是否需要架构决策 ADR？
- 是否涉及 schema 或 API？
- 是否需要拆分 Core、CLI、Agent、UI、测试任务？
- 任务是否可单独验证？
- 任务是否限定 allowed paths 和 forbidden changes？
- 是否需要安全、契约或 UI 评审？

## 常见错误

- 一次拆太大。
- 没有先冻结数据契约。
- 忽略测试任务。
- 忽略迭代关闭和知识沉淀。

## 最低通过标准

每个实现任务都必须有输入、输出、限制、验收标准和测试要求。
